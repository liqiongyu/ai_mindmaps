## Context

MMA 已在首页与编辑器内提供部分“隐私/安全承诺”与数据控制能力（如删除导图、停止分享、审计导出），但缺少集中、可访问、可被引用的信任资产页面，且缺少对关键能力（删除策略、审计导出权限）的可验证说明与测试覆盖。

本变更以最小可用的方式补齐 Trust Center 页面族（/privacy, /security, /trust），并对现有审计导出能力做“权限与内容正确”的对齐与验证。

## Goals / Non-Goals

**Goals:**

- 上线 `/privacy`、`/security`、`/trust` 三个公开页面，入口可达且内容结构清晰。
- 对用户数据保留与删除策略给出可执行指引，并与现有功能一致（删除导图、停止分享、删除后级联清理）。
- 审计导出能力保证权限边界正确（仅 owner）且内容包含必要元信息（provider/model/time/ops）。
- 为上述能力补充测试用例，确保回归可验证。

**Non-Goals:**

- 不作为法律最终文本；内容以 MVP 口径为主，后续需要法务/安全审阅。
- 不引入复杂的企业治理能力（组织/角色/审计全量导出/保留期限配置）。
- 不实现“删除账号（auth.users）”或跨系统数据删除（仅说明/覆盖本产品数据库内的用户数据）。

## Decisions

- 页面均为公开可访问（不要求登录），便于被引用与分享；涉及“如何删除/导出”的操作说明会链接到登录后的页面入口。
- 审计导出沿用现有 `/api/ai/chat/export`（已包含 owner 校验），补充测试覆盖与 Trust 页面指引。
- 统一关键口径文案（来自执行清单）：\n - `默认不将你的导图与对话用于模型训练。`\n - `你可以随时删除数据与分享链接。`

## Risks / Trade-offs

- [法律/合规准确性] → MVP 文案可能不完整。Mitigation：在页面中标注“后续将持续更新”，并在 PR 中提示需要法务审阅。
- [实现与承诺不一致] → 文案必须与真实行为一致。Mitigation：以现有功能为准撰写，并用测试覆盖关键权限边界（审计导出）。
